# YOLO Frontend Implementation Guide

This document translates the requirements in `instructions.md:1-73` plus the deployed contract artifacts under `deployments/*.json` into a concrete, extensible frontend plan. Hand it to another AI/dev and they should be able to scaffold, scale, and maintain the YOLO Protocol interface without guesswork.

---

## 1. Product Goals & Design Principles

1. **Purpose** – Deliver a fully featured, modular Web3 app that surfaces every on-chain capability shipped in YOLO Core V1 (stablecoin swaps, synthetic issuance, leveraged perps, yield flows, faucets). We must feel confident iterating on future products (loopers, strategies, more chains) without rewrites.
2. **Tone** – “Fun but elegant”: bold gradients + playful copy consistent with `yolo.wtf`, yet professional enough for institution-sized deposits. Light/dark themes with accent colors referencing the ASCII brand in `README.md`.
3. **Performance** – Instant navigation and cached state. Swapping between tabs should *not* re-fetch identical data (explicitly requested in `instructions.md:52-55`).
4. **Determinism** – Every contract call, price fetch, and conversion must be traceable to a typed service module. Never hide logic inside components.
5. **Extensibility** – Feature toggles for testnet-only modules, translations, and additional chains must be one-line additions.

## 2. Stack & Tooling Choices

| Concern | Decision | Notes |
|---------|----------|-------|
| Scaffolding | `pnpm create @rainbow-me/rainbowkit@latest` with Next.js 14 App Router + TypeScript | Satisfies “use create rainbowkit app” (`instructions.md:23`). |
| UI System | Tailwind CSS + Radix UI primitives + custom YOLO theme tokens | Easiest to theme “modern, simplistic, fun” while supporting dark/light. |
| State/Data | Wagmi v2 + TanStack Query for async caches; Zustand for ephemeral UI (drawers, slippage settings) | Keeps data fetching declarative and sharable across pages. |
| Contracts | TypeChain bindings generated from `out/` artifacts; ABIs collated under `packages/contracts` | Guarantees ABI/bytecode parity with this repo. |
| Price Streaming | Hermes client (pyth-network/hermes-client) wrapped inside `packages/hermes` + WebSocket fallback | `references/compare-intent-trade` shows how to stream. We expose a hook `useHermesPrice(symbol)`. |
| Forms & Validation | React Hook Form + Zod schemas | Ensures consistent error messaging across swap/trade/mint forms. |
| Translations | `next-intl` (or i18next) with JSON locale packs under `apps/web/locales/{en,zh-CN,zh-TW}` | Day‑1 English, Simplified, Traditional Chinese dropdown (`instructions.md:57-62`). |
| Analytics/Logs | Sentry + Wagmi transaction observers | Track failed swaps/liquidations with context. |
| Testing | Storybook for components, Playwright for flows, Vitest/Jest for hooks. |

## 3. Monorepo Layout

```
yolo-frontend/
├─ apps/
│  └─ web/                         # Next.js app generated by RainbowKit
├─ packages/
│  ├─ config/                      # Chain metadata, addresses, feature flags
│  ├─ contracts/                   # TypeChain factory exports
│  ├─ hooks/                       # Reusable wagmi/hermes/tanstack hooks
│  ├─ intl/                        # Translation helpers & locale JSON
│  ├─ services/                    # Price math, formatting, caching, analytics
│  ├─ state/                       # Zustand stores for UI + ephemeral prefs
│  └─ ui/                          # Themed components + icons/logos fallback
├─ scripts/                        # Build, generate, deployment utilities
└─ docs/                           # This file + design decisions
```

Reasons:
- Mirrors how Aave V3 structures config packages (requirement in `instructions.md:64-66`).
- Keeps contract metadata in one location so multi-chain toggles become config additions.

## 5. Global State & Data Architecture

1. **wagmi Client** – Configured with RainbowKit connectors (WalletConnect, Coinbase, Ledger, etc.). Auto-connect disabled by default to avoid phantom requests.
2. **React Query** – Each read call uses query keys of the shape `[chainId, contract, method, args]`. Examples:
   - `[84532, 'YoloOracle', 'getAssetPrice', syntheticAddress]`
   - `[84532, 'YoloHook', 'getUserTradeCount', user]`
3. **Hermes Streaming** – Wrap hermes-client into `useHermesPrice(symbol)` that:
   - Subscribes to `pythFeedId` (pull from `TradePerpConfig_84532.json`).
   - Emits `{ price, conf, exponent, publishTime }`.
   - Falls back to wagmi `readContract` if WebSocket disconnects.
4. **Zustand Stores** – Minimal stores:
   - `usePreferencesStore`: slippage, preferred fiat, language selection (mirrors Navbar dropdown), testnet toggle.
   - `useLayoutStore`: open modals, sidebar state.
5. **Derived Selectors** – Price conversions, leverage caps, etc., live in `packages/services/math`. Avoid recomputing inside components.

Caching guidelines from `instructions.md:52-55`: prefetch portfolio + price data on route change using Next.js `generateStaticParams` + Query hydration to minimize flicker.

## 7. Module Specifications (Pages)

### 7.1 Swap (`/swap`)
**Purpose**: Swap among USDC, USY, and any synthetic asset.

**Data dependencies**
- Token lists from `ConfiguredProtocol_84532.json` (`yBTC`, `yETH`, etc.).
- Real-time prices via Hermes/wagmi.
- Fee parameters from `YoloHook` (anchor vs synthetic).

**Features**
1. **Exact In / Exact Out** toggle.
2. **Price impact & fee breakdown** – compute using AMM math from `script/DeployTask_TestSwap.s.sol:45-118`.
3. **Route Builder** – Build Universal Router payloads:
   - For USDC↔USY anchor pool swap: replicate `DeployTask_TestSwap` command sequence.
   - For USY↔synthetic: call synthetic pool route (Hook handles math).
4. **Slippage control** – read from preferences store; inline validation.
5. **Preview modal** – Show expected out, min received, hook fee, price age.

**Contract interactions**
- Approve Permit2 for tokens if needed (`PERMIT2` constant from script).
- Execute Universal Router `execute` with commands (encoded via custom helper).

**UX extras**
- Display Hermes price, Oracle price, Spread. Color-coded deltas.
- Show minted supply and max caps (data from `ConfiguredProtocol_84532.json`).

### 7.2 Perp Trades (`/trade`)
Learning sources: `references/yolo-ui` (structure) + `references/compare-intent-trade` (pyth streaming) + `TradeOrchestrator.sol` methods.

**Sections**
1. **Asset selector** – Filter by tier (CRYPTO, METAL, etc. from `script/Deploy03_ConfigurePerpTrading.s.sol:92-288`).
2. **TradingView chart** – Feed Hermes price; overlay order size markers.
3. **Order ticket**:
   - Direction (Long/Short).
   - Size input (USY or synthetic).
   - Leverage slider up to `tier.maxLeverageBpsDay / 100`.
   - Borrow APR (fixedBorrowBps/100) + Funding Factor (per hour) read from `TradePerpConfig_84532.json`.
   - Spread preview (long/short).
   - Collateral requirement & liquidation threshold (10% default per instructions conversation).
4. **Position table** – Pull from `yoloHook.getUserTrade` (size, entry price, PnL, borrow/funding accrual).
5. **Actions** – Open, adjust collateral, close, claim referral rewards.

**Flow for open/close**
- Fetch price update data from Hermes -> convert to Pyth price struct (per `references/compare-intent-trade`).
- Call `TradeOrchestrator.openPosition` with `OpenPositionParams` (deadline = now + 5m).
- Display toast with tx hash, track until confirmed.

**Risk guard**
- Validate price age < config.maxPriceAgeSec (120s).
- Check user has enough USY collateral (approve if not).

### 7.3 Synthetic RWAs (`/synthetics`)

**Sections**
1. **Asset grid** – Group by sector, show mint cap, collateral factors (from `Deploy02_ConfigureProtocol.s.sol` and `ConfiguredProtocol_84532.json`).
2. **Detail drawer** – For each asset:
   - Oracle source address.
   - Supported collaterals & ratios.
   - Mint/redeem form (calls `YoloHook.mintSyntheticAsset` / `burnSyntheticAsset`).
3. **Position list** – Current minted amounts per user.

**Implementation**
- Use aggregated config JSON to auto-generate listing.
- For minting: call `IYoloHook.mintSyntheticAsset(asset, collateral, amount, minOutput, deadline)`.
- Provide collateral balance + price conversions.

### 7.4 Loop Strategies (`/strategies`)
- Placeholder card list (“Coming soon – Loop USDC→USY→…”) with CTA to join Discord.
- Keep route registered so nav is stable when we ship loops (per requirement).

### 7.5 Earn (`/earn`)
**Features**
- Deposit/withdraw USY into sUSY (anchor LP). Display APY (borrowing revenue share) by reading `StakedYoloUSD` data.
- Deposit into YLP vault (counterparty pool). Show aggregated exposure, trading volume fees.
- Historical charts (minted supply, YLP AUM).

**Implementation**
- Use wagmi to interact with `YLPVault` + `StakedYoloUSD`.
- Provide single-click “Max” buttons.

### 7.6 Dashboard (`/dashboard`)
- Consolidated balances, outstanding borrows, referral rewards, latest funding payments.
- Components reused on other pages.

### 7.7 Testnet Faucet (`/faucet`)
- Read token list from `MockAssetsAndOracles_84532.json` + `MockUSDC_84532.json`.
- Provide buttons “Mint 1,000 MockUSDC”, etc. that call `MockERC20.mint`.
- Rate-limit per wallet (frontend only) to prevent spam.

   }

3. Provide translation CLI script to validate missing keys.
4. Language selector stores preference in Zustand + `localStorage`.

## 9. Data Services & Utilities

### 9.1 Math Helpers
- Price conversions to/from `priceX8` (see `_scalePythPrice` in `script/Deploy03_ConfigurePerpTrading.s.sol:698-739`).
- Leverage/liq calculations referencing `TradeOrchestrator._computeLeverage`.
- Format numbers with `Intl.NumberFormat` respecting locale.

### 9.2 Logos & Metadata
- `packages/services/assets.ts` loads metadata from config JSON plus manual overrides (icons, display names).
- Provide fallback gradient when icon missing (per instructions).

### 9.3 Notifications
- Standard toast component for tx life-cycle.
- Display helpful copy: “Waiting for Hermes price update…”.

### 9.4 Error Handling
- Wrap wagmi calls with `try/catch` -> map to human messages.
- Example: if `TradeOrchestrator__CollateralTooSmall`, show localized string “Collateral below min 5 USY”.

## 10. Contract Integration Checklist

| Feature | Contract | Method(s) | Notes |
|---------|----------|-----------|-------|
| Swap anchor pool | Universal Router + Permit2 | `execute`, `approve` | Mirror command structure from `script/DeployTask_TestSwap.s.sol`. |
| Perp trading | `TradeOrchestrator` (`deployments/TradePerpConfig_84532.json`) | `openPosition`, `closePosition`, `topUpCollateral`, `reduceCollateral` | Provide priceUpdateData (Hermes). |
| Synthetic mint/burn | `YoloHook` | `mintSyntheticAsset`, `burnSyntheticAsset` | Collateral info from `Deploy02_ConfigureProtocol.s.sol`. |
| Earn (sUSY) | `StakedYoloUSD`, `YLPVault` | `deposit`, `withdraw`, `getExchangeRate` | Display share price. |
| Oracle data | `YoloOracle` (`src/core/YoloOracle.sol`) | `getAssetPrice`, `getSourceOfAsset` | Use only as fallback; streaming via Hermes is primary. |
| Referral Rewards | `YoloHook` | `claimReferralRewards` (if available) | Derive claimable amount from `YoloHookViews`. |
| Faucet | `MockERC20` addresses from `MockAssetsAndOracles_84532.json` | `mint` | Guard with on-screen cooldown. |

Use TypeChain to generate typed factories from `out/` to avoid manual ABI drift.

## 11. Testing & QA

1. **Unit** – Hooks/services have Vitest suites (mock wagmi). Example: `useSwapQuote` ensures math matches script output.
2. **Storybook** – Each atomic component documented with light/dark, EN/中文, disconnected/connected.
3. **E2E** – Playwright flows:
   - Connect wallet (Mocked).
   - Perform swap.
   - Open/close perp.
   - Mint synthetic.
   - Claim faucet.
4. **Smoke against Base Sepolia** – CI job runs `pnpm testnet:smoke` hitting live RPC with Tenderly fork.
5. **Accessibility** – Use `@axe-core/react` in dev to catch color contrast issues (important for funky theme).

## 13. Future-Proofing Suggestions

1. **State Machines** – For complex flows (perp opening), consider XState to model statuses (price fetch → approve → open).
2. **Fragmented Bundles** – Use Next.js route groups with suspense to lazily load heavy charts.
3. **SDK Extraction** – Move reusable Web3 logic into `packages/sdk` so third parties can integrate YOLO easily.
4. **Theme tokens** – Build on CSS variables so partner teams can white-label quickly.
5. **Metrics** – Add on-chain event listeners (e.g., `TradeAssetConfigured`) for real-time notifications if risk params change.

### TL;DR
Follow this guide to bootstrap a RainbowKit-powered, multi-chain, multilingual YOLO frontend that is modular, testable, and future-friendly. All contract bindings and configs are sourced from this repo (`deployments/*.json`, `script/*.s.sol`), Hermes streams keep prices fresh, and wagmi/query handle caching so the UX remains responsive even as we add new markets or networks. Feel free to iterate on styling, but keep the architecture layered exactly as described so additional contributors can plug into the system seamlessly.👍
